# Memanto Key-Value Storage

CAMEL integrates Memanto as a `BaseKeyValueStorage` backend. Point it at any
running Memanto server with `base_url` and `agent_id` - local or cloud is
decided by your Memanto deployment, not by CAMEL.

**References:** [Memanto docs](https://docs.memanto.ai)

---

<a id="camel.storages.key_value_storages.memanto.MemantoRESTClient"></a>

## MemantoRESTClient

```python
class MemantoRESTClient:
```

Low-level REST client used by `MemantoStorage` and `MemantoToolkit`.

**Parameters:**

- **agent_id** (str): Memanto agent identifier.
- **base_url** (str, optional): Memanto server URL. Defaults to
  `MEMANTO_BASE_URL` or `http://localhost:8000`.
- **timeout** (float, optional): HTTP timeout in seconds. (default: `30.0`)

**Environment variables:**

- `MEMANTO_BASE_URL` - server URL
- `MEMANTO_AGENT_ID` - default agent ID (used by storage/toolkit when omitted)

**Session handling:** Calls `POST /api/v2/agents/{agent_id}/activate` on init.
Memory operations send `X-Session-Token`. On `401`, the client re-activates once
and retries automatically.

### remember

```python
def remember(
    self,
    content: str,
    memory_type: MemantoMemoryType = "context",
    title: Optional[str] = None,
    confidence: float = 1.0,
    tags: Optional[List[str]] = None,
) -> str:
```

Stores a single typed memory. Returns the memory ID.

Supported `memory_type` values include: `fact`, `preference`, `goal`,
`decision`, `artifact`, `learning`, `event`, `instruction`, `relationship`,
`context`, `observation`, `commitment`, `error`.

## MemantoStorage

```python
class MemantoStorage(BaseKeyValueStorage):
```

Drop-in backend for `ChatHistoryBlock` and `ChatHistoryMemory`.

**Parameters:**

- **agent_id** (str, optional): Memanto agent ID. Falls back to `MEMANTO_AGENT_ID`.
- **base_url** (str, optional): Memanto server URL.
- **recall_limit** (int, optional): Max memories returned by `load()`. (default: `100`)
