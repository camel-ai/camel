## DockerRuntime Objects

```python
class DockerRuntime(BaseRuntime)
```

A class representing a runtime environment using Docker.
This class automatically wraps functions to be executed
in a Docker container.

**Arguments**:

- `image` _str_ - The name of the Docker image to use for the runtime.
- `port` _int_ - The port number to use for the runtime API. (default: :obj:
  `8000`)
- `remove` _bool_ - Whether to remove the container after stopping it. '
- `(default` - :obj: `True`)
- `kwargs` _dict_ - Additional keyword arguments to pass to the
  Docker client.

<a id="camel.runtime.docker_runtime.DockerRuntime.exec_run"></a>

#### exec\_run

```python
def exec_run(task: TaskConfig) -> Any
```

Run a command inside this container. Similar to `docker exec`.

**Arguments**:

- `task` _TaskConfig_ - The configuration for the task.
  

**Returns**:

- `(ExecResult)` - A tuple of (exit_code, output)
- `exit_code` - (int):
  Exit code for the executed command or `None` if
  either `stream` or `socket` is `True`.
- `output` - (generator, bytes, or tuple):
  If `stream=True`, a generator yielding response chunks.
  If `socket=True`, a socket object for the connection.
  If `demux=True`, a tuple of two bytes: stdout and stderr.
  A bytestring containing response data otherwise.
  

**Raises**:

- `RuntimeError` - If the container does not exist.
