## Purpose

OWL leverages the Model Context Protocol (MCP) to:

- **Standardize** all tool/resource calls into a single, open protocol  
- **Automate** multi‚Äëagent workflows without bespoke integrations  
- **Plug & play** new data sources and services with minimal configuration  

---

## 1. Configuring MCP Servers

Create a file `mcp_servers_config.json` at your project root:

```json
{
  "mcpServers": {
    "desktop-commander": {
      "command": "npx",
      "args": ["-y", "@wonderwhy-er/desktop-commander"]
    },
    "playwright": {
      "command": "npx",
      "args": ["-y", "@executeautomation/playwright-mcp-server"]
    },
    "fetch": {
      "command": "python",
      "args": ["-m", "mcp_server_fetch"]
    }
  }
}
```

- `desktop-commander`: control local applications  
- `playwright`: browser automation  
- `fetch`: HTTP‚Äëbased content retrieval  

OWL will **discover** and **index** these servers at startup.

---

## 2. Initializing the MCP Toolkit

```jsx
import { Path } from 'path';
import { MCPToolkit } from 'mcp_toolkit';

const configPath = Path.join(__dirname, 'mcp_servers_config.json');
const mcpToolkit = new MCPToolkit({ configPath });

// Establish connections to all configured servers
await mcpToolkit.connect();
```

- Using `Path.join` ensures cross‚Äëplatform paths  
- `connect()` asynchronously initializes each MCP client
